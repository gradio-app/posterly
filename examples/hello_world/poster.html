What this is

posterly is a Claude Code skill that builds academic conference posters as a single HTML/CSS file rendered to print-ready PDF via headless Chromium. You iterate by measuring, not eyeballing — the screen preview lies; only emulate_media("print") at the exact viewport tells the truth.

How the gates work
  • preflight — static lint: LaTeX residue, raw < inside math, missing local images, role validation.
  • measure — print-emulated geometry: column-bottom spread (< 5 px), gap to next strip ([30, 50] px), poster bbox aligns to page within ±2 px.
  • polish — soft visual checks: figure-AR sizing, broken images, typography orphans, space-between fill.
  • verify-final — pdfinfo-based PDF sanity: page count, dimensions, file size.
Why measure, not eyeball

The screen preview lies: @media print uses a different unit scale, web fonts substitute, and a column that looks aligned on screen can sit 20 px off in the printed PDF. posterly renders at the exact print viewport and reports numbers — so “looks aligned” becomes spread = 0.0 px, or it doesn't.

  • Fill, don't stretch. A card padded out with flex: 1 clears the bottom-edge gate while half of it is blank — balance it with real content, never whitespace.
  • Autocrop figures so the image fills its card instead of floating in a gray margin.
  • Guard trailing glyphs (×, ↑, %) with white-space: nowrap so a lone symbol never orphans onto its own line.
What's in the box
  • templates/ — portrait 2-column, landscape 4-column, and hero gallery, each pre-wired with the measurement roles.
  • tools/ — the four-gate poster_check.py CLI plus a standalone render_preview.py.
  • examples/ — this file: the smallest poster that clears every gate, your install check.
