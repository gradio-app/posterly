What this is

posterly is a Claude Code skill that builds academic conference posters as a single HTML/CSS file rendered to print-ready PDF via headless Chromium. You iterate by measuring, not eyeballing — the screen preview lies; only emulate_media("print") at the exact viewport tells the truth.

This hello-world example demonstrates every measurement role the gates look for: poster, header, body, column, card, footer-strip, footer.

How the gates work
  • preflight — static lint: LaTeX residue, raw < inside math, missing local images, role validation.
  • measure — print-emulated geometry: column-bottom spread (target < 5 px) and gap to next strip (target [30, 50] px).
  • polish — soft visual checks: figure-AR sizing, typography orphans, space-between fill.
  • verify-final — pdfinfo-based PDF sanity: page count, dimensions, file size.
Customize

Edit the :root design tokens for colors and fonts; replace the inline QR placeholder with a real QR generated by qrencode or Python’s qrcode; drop figures into a sibling images/ directory.

For multi-column posters use a template from templates/. The same gates run there too.
